DiaScript 프로젝트 개발 및 디버깅 일지

작성일자: 2026-05-17
작성자: Hyunchan An

1. 주요 아키텍처 수정 및 문제 해결 내역

1.1. GPU 가속(NVIDIA RTX 5080) 병목 현상 완수 해결
- 현상: CUDAExecutionProvider가 비활성화되어 CPU 사용량이 100%에 달하고 GPU가 과소 점유되는 Silent Fallback 문제 발생.
- 원인: 내 윈도우 환경에서 CPU 전용 onnxruntime 패키지와 GPU용 onnxruntime-gpu 패키지가 충돌하여 경로가 꼬였음을 확인.
- 조치: requirements.txt를 정밀 수정하고 충돌 패키지를 강제 언인스톨한 후, onnxruntime-gpu 단독 재배포를 완료하여 GPU 점유율 94%, CPU 4%의 강력한 하드웨어 가속 성능을 확보함.

1.2. 윈도우 파일 잠금(Permission Error) 원천 차단 설계
- 현상: 브라우저 캐시가 spk_01_sample.wav 파일을 독점 읽기 락으로 잡고 있어, 화자 분리 재연산 시 sf.write 덮어쓰기 권한 거부 에러 유발.
- 조치: 오디오 조각 생성 시 밀리초 단위 고유 타임스탬프 해시(int(time.time() * 1000))를 결합한 가변 유니크 파일명 체계로 전환하여 파일 쓰기 충돌을 원천 차단함.

1.3. Gradio 비동기 체인 꼬임 및 KeyError: 9 문제 해결 (단일 파이프라인 통합)
- 현상: perform_diarization과 update_speaker_ui_list를 .then() 체인으로 순차 호출할 때 Gradio 엔진이 내부 가상 세션에 컴포넌트 값을 복사(deepcopy)하려 시도하다 비동기 Future 객체 직렬화 충돌 및 KeyError: 9 서버 크래시를 유발함.
- 조치: 두 비동기 함수를 단 하나의 파이썬 함수인 run_diarization_pipeline 및 run_recluster_pipeline으로 전격 단일화함. 이로써 Gradio 엔진이 중간 세션값을 강제 복사해 서빙할 필요를 배제하여 안정성을 극대화함.

1.4. Gradio 윈도우 경로 해싱 파싱 버그 완화 핫픽스
- 현상: 임시 WAV 파일 경로의 백슬래시(\)로 인해 Gradio 리소스 이송기가 경로를 오인하고 최상위 내 작업 폴더(E:\Github\DiaScripter) 자체를 바이너리로 open하려다 Errno 13 권한 거부 에러 유발.
- 조치: app.py의 모든 파일 입출력 반환 경로에 .replace('\\', '/') 를 일괄 적용하여 POSIX 규격으로 표준 정화 처리함.
- 추가 조치: 비활성화 처리해야 하는 8인 외의 오디오 상자에 value=None 속성을 쏘아 보내면 Gradio가 빈 문자열에 대한 abspath를 강제 연산하여 다시 CWD 폴더 읽기를 시도하는 버그를 체포함. 이에 따라 비활성화 컴포넌트는 value 업데이트를 전면 생략하고 오직 gr.update(visible=False) 처럼 가시성만 제어하도록 무필드 가시성 패치(Zero-Value Visibility Patch)를 적용함.

2. 미결 과제 및 디버깅 강화 계획

2.1. 윈도우 환경 Gradio 파일 캐시 가상 시스템 무결성 디버깅 강화
- 현상: 상기 핫픽스 적용에도 불구하고 윈도우 파일 시스템 특성과 Starlette/FastAPI 캐시 기법 간의 마찰로 인해 여전히 간헐적인 디렉토리 접근 Permission denied(Errno 13)가 감지되고 있음.
- 향후 디버깅 강화 계획: 
  - app.py 내의 주요 비즈니스 함수들에 상세 traceback 추적 로그 기법을 심어 에러가 발생한 절대적인 파일 라인 넘버를 UI에 그대로 인쇄하도록 구성 완료.
  - 추가적으로, Gradio가 내부 파일 이송 작업 도중 open('E:\Github\DiaScripter', 'rb') 을 시도하는 Starlette 미들웨어 소스코드 내부 동작에 대한 정밀 프로파일링을 진행하여 강제 바이패스 가드를 신설할 예정.

2.2. 오디오 플레이어 다중 렌더링 메소드 통째 전환 검토 (gr.HTML 마이그레이션)
- 목적: 8쌍의 gr.Audio 및 gr.Textbox 다중 바인딩 명세가 Gradio 파일 캐싱 구조와 잦은 충돌을 빚는 근본 한계를 극복.
- 내 계획: 최대 8쌍의 개별 컴포넌트 대신 단 하나의 gr.HTML 컴포넌트만을 화면에 배치하고, 내 파이썬 Flask/FastAPI 엔드포인트를 통해 다이렉트로 오디오 파일을 스트리밍 서빙하도록 메소드를 통째로 전환하는 마이그레이션 수행 검토. 이 경우 Gradio의 고유 파일 캐싱 및 복사 메커니즘을 완전히 우회하여 윈도우 환경에서의 디스크 권한 에러를 100% 원천 종식시킬 수 있음.

2.3. 대용량 장시간 SRT 자막 정합성 검증 및 예외 분석
- 실제 2~3시간 분량의 극장판 음원에 대해 최종 3단계 전사 버튼이 기동될 때의 자막 싱크 틀어짐이나 예외 상황에 대한 실시간 모니터링 체계 고도화.
